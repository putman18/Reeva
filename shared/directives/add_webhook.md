# Add Webhook Directive

## Purpose

Register a new Modal webhook that triggers an autonomous agent to execute a specific directive when called via HTTP POST.

## When to use

When the user says "add a webhook that..." or "create an endpoint that..."

## Inputs

- What the webhook should do (natural language description)
- What data it will receive in the POST body
- What tools it needs (default: send_email, read_sheet, update_sheet)

## Steps

1. **Create the directive file** in `directives/`
   - Name it descriptively, e.g. `directives/process_new_lead.md`
   - Write clear instructions: goal, inputs, tools to call, outputs, edge cases
   - Use the payload fields by name so the agent knows what to expect

2. **Register in webhooks.json** by adding an entry to the `webhooks` array:
   ```json
   {
     "slug": "process-new-lead",
     "directive": "process_new_lead.md",
     "description": "Processes a new lead from the CRM form",
     "tools": ["send_email", "update_sheet"]
   }
   ```
   - `slug` becomes the URL path: `POST /process-new-lead`
   - `tools` scopes what the agent can use (keep it minimal)

3. **Start the local server** (if not already running):
   ```bash
   python execution/webhook_server.py
   # or on a custom port:
   python execution/webhook_server.py --port 8888
   ```
   Server runs at `http://localhost:8000` by default.

4. **Test the endpoint**:
   ```bash
   curl -X POST http://localhost:8000/<slug> \
     -H "Content-Type: application/json" \
     -d '{"field": "value"}'
   ```

5. **Verify** the response includes `"status": "ok"` and a meaningful result.

## Directive file template

```markdown
# <Webhook Name> Directive

## Goal
<What this webhook accomplishes>

## Input payload fields
- `field_name`: description

## Steps
1. <Step one>
2. <Step two>

## Tools
- `send_email`: use when...
- `update_sheet`: use when...

## Edge cases
- If X, then Y
```

## Available tools

| Tool | Script | Purpose |
|------|--------|---------|
| send_email | execution/send_email.py | Send email via SMTP |
| read_sheet | execution/read_sheet.py | Read rows from Google Sheet |
| update_sheet | execution/update_sheet.py | Write rows to Google Sheet |

## Notes

- Only grant tools the webhook actually needs (principle of least privilege)
- Environment variables are read from `.env` at server startup
- Google OAuth tokens must be pre-generated by running read_sheet.py or update_sheet.py once interactively before starting the server
- The server reloads `webhooks.json` on every request, so you can add webhooks without restarting
- To expose the server to the internet (e.g. for external services to call it), use `ngrok http 8000`
